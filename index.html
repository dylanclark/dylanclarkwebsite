About this site

TL;DR: I use this repo as a framework and Netlify to host.

Background

At the risk of writing too much fluff and sounding like a recipe website, I figured some of this info would be good to share for anyone who's curious to create their own personal site.

I have spent a few years trying to nail down the right configuration for for a personal website. I've gone through a few tools, finding some too expensive (this one is free) and some too complicated (React for a personal site...).

I love web1.0: simple, fast, javascript-free websites without a care in the world for SEO or the like. I'm tired of the modern internet where simple websites require social sign-on and I can't pay rent without opening some form of Chromium.

I came across this sample site for 'John Doe' that is implemented in a single HTML file with a CSS stylesheet and knew it's what I wanted for myself. It has no scripts and no dependencies, I can edit it with any text editor, and it's fast. The github repo I used as a template is here.

My repo for the site is here. I've changed some colors (and the content obviously) but otherwise used only what came with the template.

I then found the Netlify free tier to be an excellent hosting/deployment/DNS service. It's linked to the GitHub repo, so a simple 'git push' to the master branch of my repo and the new edits are live!

Steps to deploy your own site

  • Fork or copy the template from github (https://github.com/cadars/john-doe)
  • Edit the HTML and CSS as you see fit
  • Create a free Netlify account here
  • Link your Netlify and GitHub accounts (instructions here)
  • [optional] Connect any custom domain to netlify (instructions here)
  • git commit -am "my website is done!"
  • git push
